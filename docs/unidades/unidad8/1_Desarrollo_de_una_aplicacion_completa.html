Desarrollo de una Aplicación Completa

En esta sección, vamos a desarrollar una aplicación completa utilizando Ruby on Rails. La aplicación será un sistema de gestión de usuarios con las siguientes funcionalidades:

Crear un nuevo usuario.
Mostrar un usuario.
Editar un usuario.
Eliminar un usuario.

Para desarrollar la aplicación, vamos a seguir los siguientes pasos:

Crear una nueva aplicación de Rails.
Crear un modelo de usuario.
Crear un controlador de usuarios.
Crear vistas para las acciones de la aplicación.
Configurar las rutas de la aplicación.
Ejecutar la aplicación.

Vamos a empezar por crear una nueva aplicación de Rails.

Crear una Nueva Aplicación de Rails

Para crear una nueva aplicación de Rails, vamos a utilizar el siguiente comando:

rails new user_management

Este comando creará una nueva aplicación de Rails llamada user_management en un directorio con el mismo nombre. Una vez que la aplicación se haya creado, vamos a cambiar al directorio de la aplicación:

cd user_management

Crear un Modelo de Usuario

Para crear un modelo de usuario, vamos a utilizar el siguiente comando:

rails generate model User name:string email:string

Este comando creará un nuevo modelo de usuario con dos atributos: name y email. Una vez que el modelo se haya creado, vamos a ejecutar la migración para crear la tabla de usuarios en la base de datos:

rails db:migrate

Crear un Controlador de Usuarios

Para crear un controlador de usuarios, vamos a utilizar el siguiente comando:

rails generate controller Users

Este comando creará un nuevo controlador de usuarios con las acciones index, show, new, edit, create, update y destroy. Una vez que el controlador se haya creado, vamos a abrir el archivo app/controllers/users_controller.rb y agregar los métodos correspondientes a cada acción:

class UsersController < ApplicationController
  def index
    @users = User.all
  end

  def show
    @user = User.find(params[:id])
  end

  def new
    @user = User.new
  end

  def edit
    @user = User.find(params[:id])
  end

  def create
    @user = User.new(user_params)

    if @user.save
      redirect_to @user
    else
      render 'new'
    end
  end

  def update
    @user = User.find(params[:id])

    if @user.update(user_params)
      redirect_to @user
    else
      render 'edit'
    end
  end

  def destroy
    @user = User.find(params[:id])
    @user.destroy

    redirect_to users_path
  end

  private
    def user_params
      params.require(:user).permit(:name, :email)
    end
end

En este código, hemos definido los métodos index, show, new, edit, create, update y destroy para el controlador de usuarios. Cada método realiza una acción específica en la aplicación, como mostrar una lista de usuarios, mostrar un usuario, crear un nuevo usuario, editar un usuario, actualizar un usuario y eliminar un usuario.

Crear Vistas para las Acciones de la Aplicación

Para crear vistas para las acciones de la aplicación, vamos a abrir el archivo app/views/users/index.html.erb y agregar el siguiente código:

<h1>Users</h1>

<table>
  <tr>
    <th>Name</th>
    <th>Email</th>
    <th></th>
    <th></th>
    <th></th>
  </tr>

  <% @users.each do |user| %>
    <tr>
      <td><%= user.name %></td>
      <td><%= user.email %></td>
      <td><%= link_to 'Show', user %></td>
      <td><%= link_to 'Edit', edit_user_path(user) %></td>
      <td><%= link_to 'Destroy', user, method: :delete, data: { confirm: 'Are you sure?' } %></td>
    </tr>
  <% end %>
</table>

<br>

<%= link_to 'New User', new_user_path %>

Este código define la vista index para mostrar una lista de usuarios en la aplicación. La vista incluye una tabla con las columnas Name, Email y Acciones, que permiten al usuario ver, editar y eliminar un usuario. También incluye un enlace para crear un nuevo usuario.

A continuación, vamos a crear las vistas show, new, edit y _form para las acciones de la aplicación. Para crear la vista show, vamos a abrir el archivo app/views/users/show.html.erb y agregar el siguiente código:

<p>
  <strong>Name:</strong>
  <%= @user.name %>
</p>

<p>
  <strong>Email:</strong>
  <%= @user.email %>
</p>

<%= link_to 'Edit', edit_user_path(@user) %> |
<%= link_to 'Back', users_path %>

Este código define la vista show para mostrar un usuario en la aplicación. La vista incluye los atributos Name y Email del usuario, así como enlaces para editar el usuario y volver a la lista de usuarios.

Para crear la vista new, vamos a abrir el archivo app/views/users/new.html.erb y agregar el siguiente código:

<h1>New User</h1>

<%= render 'form' %>

<%= link_to 'Back', users_path %>

Este código define la vista new para crear un nuevo usuario en la aplicación. La vista incluye un formulario para introducir el nombre y el email del usuario, así como un enlace para volver a la lista de usuarios.

Para crear la vista edit, vamos a abrir el archivo app/views/users/edit.html.erb y agregar el siguiente código:

<h1>Edit User</h1>

<%= render 'form' %>

<%= link_to 'Show', @user %> |
<%= link_to 'Back', users_path %>

Este código define la vista edit para editar un usuario en la aplicación. La vista incluye un formulario para modificar el nombre y el email del usuario, así como enlaces para ver el usuario y volver a la lista de usuarios.

Por último, vamos a crear la vista _form para el formulario de usuario. Para crear la vista _form, vamos a abrir el archivo app/views/users/_form.html.erb y agregar el siguiente código:

<%= form_with(model: @user, local: true) do |form| %>
  <% if @user.errors.any? %>
    <div id="error_explanation">
      <h2><%= pluralize(@user.errors.count, "error") %> prohibited this user from being saved:</h2>

      <ul>
        <% @user.errors.full_messages.each do |message| %>
          <li><%= message %></li>
        <% end %>
      </ul>
    </div>
  <% end %>

  <div class="field">
    <%= form.label :name %>
    <%= form.text_field :name %>
  </div>

  <div class="field">
    <%= form.label :email %>
    <%= form.text_field :email %>
  </div>

  <div class="actions">
    <%= form.submit %>
  </div>
<% end %>

Este código define el formulario de usuario para introducir el nombre y el email del usuario en la aplicación. El formulario incluye campos para el nombre y el email del usuario, así como un botón para guardar los cambios.

Configurar las Rutas de la Aplicación

Para configurar las rutas de la aplicación, vamos a abrir el archivo config/routes.rb y agregar las siguientes rutas:

Rails.application.routes.draw do
  resources :users

  root 'users#index'
end

Estas rutas definen las acciones de la aplicación para el controlador de usuarios, así como la ruta raíz de la aplicación. La ruta raíz se establece en la acción index del controlador de usuarios, que muestra una lista de usuarios en la aplicación.

Ejecutar la Aplicación

Para ejecutar la aplicación, vamos a utilizar el siguiente comando:

rails server

Este comando iniciará el servidor de Rails y la aplicación estará disponible en http://localhost:3000. Una vez que la aplicación esté en funcionamiento, podremos crear, mostrar, editar y eliminar usuarios en la aplicación.

Conclusiones

En este tutorial, hemos desarrollado una aplicación completa utilizando Ruby on Rails. La aplicación es un sistema de gestión de usuarios con las funcionalidades de crear, mostrar, editar y eliminar usuarios. Hemos creado un modelo de usuario, un controlador de usuarios, vistas para las acciones de la aplicación y configurado las rutas de la aplicación. Finalmente, hemos ejecutado la aplicación y probado las funcionalidades implementadas.

Ruby on Rails es un marco de desarrollo web potente y flexible que facilita la creación de aplicaciones web completas y funcionales. Con Rails, podemos desarrollar aplicaciones de forma rápida y eficiente, utilizando las mejores prácticas de desarrollo web y aprovechando las ventajas de Ruby como lenguaje de programación. Si estás interesado en aprender más sobre Ruby on Rails, te recomiendo que explores la documentación oficial y los recursos de aprendizaje disponibles en línea. ¡Buena suerte con tus proyectos de desarrollo web!
